Probability Foundations
The mathematical foundation for statistics

What is Probability?

Probability is the mathematical language of uncertainty. It provides the framework for quantifying how likely events are to occur, which forms the foundation for statistical inference. While statistics draws conclusions from data, probability tells us what we should expect to see under given conditions.

Why Engineers Need Probability:
  • Quality Control: Determining acceptable defect rates
  • Reliability Engineering: Predicting system failure times
  • Signal Processing: Filtering noise from measurements
  • Risk Assessment: Evaluating safety margins in design
  • Queueing Theory: Optimizing service systems

Experiments, Sample Space, and Events

🔬 Experiment

An experiment is any process that generates well-defined outcomes. In engineering, experiments might include:

  • Testing a manufactured component for defects
  • Measuring the breaking strength of a material
  • Recording the number of arrivals at a service station

📊 Sample Space (S)

The sample space is the set of all possible outcomes of an experiment. Each outcome is called a sample point.

Engineering Examples:
ExperimentSample Space
Testing a single electronic componentS = {Pass, Fail}
Measuring voltage (0-15V continuous)S = {x | 0 ≤ x ≤ 15 volts}
Customers arriving in an hourS = {0, 1, 2, 3, ...}
Classifying concrete strengthS = {Low, Medium, High}
Temperature reading (°C)S = (-∞, ∞) or bounded range

🎯 Event

An event is a subset of the sample space — a collection of outcomes. Events are typically denoted by capital letters (A, B, C).

Example: For customer arrivals S = {0,1,2,3,...}
  • Event A = "at least 3 customers arrive" = {3, 4, 5, ...}
  • Event B = "fewer than 2 customers arrive" = {0, 1}
  • Event C = "exactly 5 customers arrive" = {5}

Engineering Example: Circuit Reliability

A circuit has two components A and B. Let:

  • Event A = "Component A fails" (P(A) = 0.1)
  • Event B = "Component B fails" (P(B) = 0.15)
  • Both fail simultaneously with probability 0.03

Find:

  • P(A ∪ B) = P(A) + P(B) - P(A∩B) = 0.1 + 0.15 - 0.03 = 0.22
  • P(A' ∩ B') = 1 - P(A ∪ B) = 0.78 (system works)

Probability Rules

📐 Axioms of Probability

  1. Non-negativity: P(A) ≥ 0 for any event A
  2. Normalization: P(S) = 1
  3. Additivity: For disjoint events, P(A ∪ B) = P(A) + P(B)

Addition Rule

P(A ∪ B) = P(A) + P(B) - P(A ∩ B)

If A and B are disjoint: P(A ∩ B) = 0, so P(A ∪ B) = P(A) + P(B)

Complement Rule

P(A') = 1 - P(A)

P(A) + P(A') = 1

🏭 Engineering Application: Quality Control

A manufacturing line produces items with two types of defects:

  • P(surface defect) = 0.08
  • P(structural defect) = 0.05
  • P(both defects) = 0.02

P(at least one defect) = 0.08 + 0.05 - 0.02 = 0.11

P(no defects) = 1 - 0.11 = 0.89

Conditional Probability

P(A|B) = P(A ∩ B) / P(B), provided P(B) > 0

The probability of A given that B has occurred.

🔄 Disjoint Events

A ∩ B = ∅ ⇒ P(A|B) = 0

If A and B cannot occur together, knowing B occurred means A cannot occur.

⚡ Independent Events

P(A|B) = P(A) ⇔ P(A ∩ B) = P(A)P(B)

Knowing B gives no information about A.

🔌 Engineering Example: System Reliability

A system has two components in series. System fails if either fails.

  • P(A fails) = 0.1, P(B fails) = 0.2
  • Failures are independent

P(system fails) = P(A fails ∪ B fails)

= 0.1 + 0.2 - (0.1)(0.2) = 0.28

P(B fails | system fails) = P(B ∩ system fails)/P(system fails)

= P(B fails)/0.28 = 0.2/0.28 ≈ 0.714

Total Probability & Bayes' Theorem

📊 Law of Total Probability

If events B₁, B₂, ..., Bₖ partition the sample space:

P(A) = Σ P(A|Bᵢ) × P(Bᵢ)

🧮 Bayes' Theorem (2 events)

P(B|A) = P(A|B) × P(B) / P(A)

where P(A) = P(A|B)P(B) + P(A|B')P(B')

🧮 Bayes' Theorem (k events)

P(Bⱼ|A) = P(A|Bⱼ)P(Bⱼ) / Σ P(A|Bᵢ)P(Bᵢ)

Random Variables

A random variable is a function that assigns a numerical value to each outcome in the sample space.

📊 Discrete Random Variable

Takes countable values (often integers)

PMF: P(X = x) = p(x)

Requirements: p(x) ≥ 0, Σ p(x) = 1

📈 Continuous Random Variable

Takes any value in an interval

PDF: f(x) ≥ 0, ∫ f(x)dx = 1

P(a ≤ X ≤ b) = ∫ f(x)dx

📐 Mean, Variance, Standard Deviation

MeasureFormula (Discrete)
Mean μ = E[X]Σ x·p(x)
Variance σ² = Var(X)Σ (x-μ)²·p(x) = E[X²] - (E[X])²
SD σ√Var(X)

🔧 Engineering Example: Number of Defects

Let X = number of defects in a batch of 5 items, with distribution:

x0123
p(x)0.70.20.080.02

E[X] = 0×0.7 + 1×0.2 + 2×0.08 + 3×0.02 = 0 + 0.2 + 0.16 + 0.06 = 0.42 defects

E[X²] = 0²×0.7 + 1²×0.2 + 2²×0.08 + 3²×0.02 = 0 + 0.2 + 0.32 + 0.18 = 0.70

Var(X) = E[X²] - (E[X])² = 0.70 - (0.42)² = 0.70 - 0.1764 = 0.5236

σ = √0.5236 ≈ 0.724 defects

Properties of Mean and Variance

  • E[aX + b] = a·E[X] + b
  • Var(aX + b) = a²·Var(X)
  • E[X + Y] = E[X] + E[Y] (always)
  • If X, Y independent: Var(X + Y) = Var(X) + Var(Y)
  • E[XY] = E[X]E[Y] if independent

Special Distributions in Engineering

📊 Discrete Distributions

Binomial(n, p)

P(X=k) = C(n,k) pᵏ (1-p)ⁿ⁻ᵏ

Mean: np, Variance: np(1-p)

🔧 Engineering Applications:
  • Quality Control: Number of defective items in a sample of n products from a production line with defect probability p
  • Reliability: Number of successful operations out of n identical components in a system
  • Communications: Number of correctly transmitted bits in a message of n bits with bit error rate p
  • Manufacturing: Number of conforming parts in a batch of n when process capability is p
  • Acceptance Sampling: Probability of accepting a lot based on number of defective items in a sample
Example: In a semiconductor fab, 5% of chips are defective. In a batch of 20 chips, the probability of exactly 2 defects is C(20,2)(0.05)²(0.95)¹⁸ ≈ 0.189
Geometric(p)

P(X=k) = (1-p)ᵏ⁻¹ p, k=1,2,3,...

Mean: 1/p, Variance: (1-p)/p²

🔧 Engineering Applications:
  • Reliability Testing: Number of tests until first component failure
  • Quality Control: Number of items inspected until finding the first defective
  • Communication Networks: Number of transmission attempts until successful packet delivery
  • Maintenance: Number of days until a machine requires repair
  • Safety Engineering: Number of operations until a safety device activates
Example: A machining operation has a 2% chance of producing a non-conforming part. The probability that the first non-conforming part occurs on the 10th piece is (0.98)⁹(0.02) ≈ 0.0167
Poisson(λ)

P(X=k) = e⁻λ λᵏ / k!

Mean: λ, Variance: λ

🔧 Engineering Applications:
  • Manufacturing: Number of surface defects per unit area (paint, coating, semiconductor wafers)
  • Traffic Engineering: Number of vehicles arriving at an intersection per minute
  • Telecommunications: Number of calls arriving at a call center per hour
  • Reliability: Number of system failures per month
  • Environmental Engineering: Number of contaminant particles per liter of water
  • Software Engineering: Number of bugs per thousand lines of code
  • Textile Engineering: Number of flaws per meter of fabric
  • Nuclear Engineering: Number of radioactive decay events per second
Example: A fiber optic cable has an average of 2 flaws per kilometer. The probability of exactly 3 flaws in a 2-kilometer segment is e⁻⁴ × 4³/3! ≈ 0.195 (λ = 2×2 = 4)
Uniform Discrete (a,b)

P(X=k) = 1/(b-a+1)

Mean: (a+b)/2, Variance: ((b-a+1)²-1)/12

🔧 Engineering Applications:
  • Digital Systems: Random number generation in computers
  • Simulation: Generating random integers for Monte Carlo studies
  • Quality Control: Random selection of items for inspection
  • Cryptography: Generating random keys within a range
  • Scheduling: Random assignment of jobs to machines
Example: A random number generator produces integers from 1 to 100 with equal probability. P(X = 42) = 1/100 = 0.01

📈 Continuous Distributions

Uniform Continuous (a,b)

f(x) = 1/(b-a) for a ≤ x ≤ b

Mean: (a+b)/2, Variance: (b-a)²/12

🔧 Engineering Applications:
  • Manufacturing: Rounding errors in measurements
  • Signal Processing: Phase of a random signal
  • Simulation: Generating random numbers for Monte Carlo methods
  • Quality Control: Distribution of fill volumes when machine is perfectly centred
  • Civil Engineering: Random orientation of aggregate particles in concrete
  • Electrical Engineering: Quantization error in analog-to-digital conversion
Example: An ADC rounds to the nearest 0.1V. The quantization error is uniformly distributed between -0.05V and +0.05V. P(error < 0.02V) = (0.07)/0.1 = 0.7
Normal(μ, σ²)

f(x) = 1/(σ√(2π)) e^{-(x-μ)²/(2σ²)}

Mean: μ, Variance: σ²

68-95-99.7 rule

🔧 Engineering Applications:
  • Manufacturing: Distribution of part dimensions (diameters, lengths, thicknesses) around target value
  • Quality Control: Process capability analysis (Cpk, Cp indices)
  • Civil Engineering: Concrete strength variations
  • Mechanical Engineering: Fatigue life distribution (log-normal often, but normal for many properties)
  • Electrical Engineering: Thermal noise voltage in resistors
  • Chemical Engineering: Purity variations in chemical processes
  • Environmental Engineering: Pollutant concentration variations
  • Structural Engineering: Load variations on structures
  • Geotechnical Engineering: Soil property variations
  • Aerospace: Thrust variations in rocket engines
Example: A shaft diameter is normally distributed with μ = 50.00 mm and σ = 0.02 mm. The probability of a shaft being within tolerance (49.97 to 50.03 mm) is approximately 0.866 (within ±1.5σ)
